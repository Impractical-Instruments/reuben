# V1.5 Stereo output + `pan` op — human testing

A hands-on walkthrough of the first V1.5 deliverable ([ADR-0026](adr/0026-v1-finish-line-osc-out-and-stereo.md)):
**stereo output** and a new **`pan` operator**. Until now reuben's master was a single mono
buffer fanned to every device channel — indefensible for a modern audio engine. Now the master
is **N logical channels**, and any tap can be pinned to one of them.

The model (ADR-0026): the Signal stays **mono — one channel per edge**. "Stereo" is just **two
edges** landing on two master channels, not a wider Signal. So nothing in the 17 existing
operators changed. A master tap (an `outputs` entry) gains an optional **`channel`** index:

- **omitted** → broadcast to every channel (the historical mono fan; existing instruments are
  bit-identical),
- `channel: 0` → left only, `channel: 1` → right only, and so on.

Master width is **derived from the instrument** (highest referenced channel + 1, floored to
stereo), never from the device. `audio.rs` owns the **logical→device map** — straight copy when
counts match, downmix for a mono device, zero-fill for a device with more channels.

The new **`pan`** op is the workhorse: one mono `audio` in → `left` / `right` out, **equal-power**
(−3 dB center, constant perceived loudness across the sweep). Pan position is a **Signal input**
with a param default (ADR-0017 one-port-one-type), so an LFO can auto-pan or a knob can park it.
The two outputs are tapped directly as `channel: 0` / `channel: 1` — the `output` op is vestigial
for a stereo patch.

## Setup

**Run the stereo example** (opens your default audio device, listens on UDP `0.0.0.0:9000`):

```sh
cargo run -p reuben-native --bin reuben -- play instruments/stereo-autopan.json
```

Use **headphones** or speakers with real left/right separation — the whole point is hearing
position.

## 1. Auto-pan — `stereo-autopan.json`

A playable 8-voice saw synth whose voice is swept across the stereo field by an LFO driving the
`pan` op. Send notes over OSC (any OSC source — TouchOSC, Max, a script, `oscsend`):

```
/voicer/notes  [69.0, 1.0]    # note-on  (MIDI 69 = A4, gate 1)
/voicer/notes  [69.0, 0.0]    # note-off (gate 0)
```

Play a sustained note (or a chord — send several note-ons) and listen: the tone should glide
**left ↔ right** about once every two seconds.

Tweak it live:

- `/autopan/rate  [0.5]` — sweep speed in Hz. Push it up (`[3.0]`) for a fast tremolo-pan; set it
  near zero (`[0.01]`) to park the position.
- `/autopan/depth [1.0]` — how far it swings (0 = dead center, 1 = hard L↔R).

**Park-and-place by hand:** set `/autopan/depth [0.0]` (or `/autopan/rate [0.0]`) so the LFO
holds at center, then drive the pan position directly: `/pan/pan [-1.0]` hard-left, `[0.0]`
center, `[1.0]` hard-right. This is the unwired-default param taking over when the swing is flat.

> Equal-power check: at center (`/pan/pan [0.0]`) each side sits ~3 dB below a hard-panned side,
> but the *total* loudness is constant as you sweep — no dip in the middle, no bulge at the edges.

## 2. What "bit-identical for mono" means

Every existing instrument still has mono taps (no `channel`), so it broadcasts to both channels.
Run the default rig and a mono device gets exactly the old samples back:

```sh
cargo run -p reuben-native --bin reuben -- play instruments/default.json
```

It sounds identical to before — stereo cost nothing for instruments that don't ask for it. This
is asserted in the tests (`crates/reuben-core/tests/stereo_pan.rs`,
`broadcast_instrument_is_bit_identical_mono_and_across_channels`).

## 3. Roll your own stereo patch

Tap two sources to two channels by hand — e.g. two detuned oscillators hard-panned:

```json
"outputs": [
  { "node": "/oscL", "port": "audio", "channel": 0 },
  { "node": "/oscR", "port": "audio", "channel": 1 }
]
```

Or feed one voice through `pan` and tap its `left`/`right` as `channel: 0` / `1` (what
`stereo-autopan.json` does). Validate before playing:

```sh
cargo run -p reuben-native --bin reuben -- validate instruments/your-patch.json
```

## What this proves

- A `pan` op places mono audio in the stereo field with an equal-power law.
- Master taps route to independent logical channels; width is derived from the instrument.
- Mono instruments are untouched and bit-identical.
- `audio.rs` maps logical channels onto whatever the device offers.

Still open in V1.5 (separate work): **OSC-out** and the **Windows build + release workflow**.
Windows stereo, like all Windows audio, needs a manual smoke pass on real hardware (CI can't see
WASAPI device init) — see [ADR-0026](adr/0026-v1-finish-line-osc-out-and-stereo.md).
