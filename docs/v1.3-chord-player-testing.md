# V1.3 Toy — the Chord player: human testing

A hands-on walkthrough of **Toy 2 of 3** in the V1.3 "Toys" release
([ADR-0022](adr/0022-the-toys.md)): a tap-to-play diatonic harmony instrument. Seven buttons are
the diatonic triads **I–vii°** of the current key; tap-and-hold sustains a chord, release stops
it. A key selector makes held and tapped chords **re-spell live** on a key change — reuben's
signature, and the reason this Toy exists over a fixed-progression pad.

This follows the V1.4 control-surface pattern (`docs/v1.4-control-surface-testing.md`): the
instrument is the source of truth, a generated **Hexler TouchOSC** `.tosc` is the surface you
hold. If `cargo test` *proves* the engine works, this is how you *play* the Chord player.

## What's under the hood

```
7 chord buttons ─/chord/set [degree,gate]→ chord ─degree→ voicer(12) ─freq→ osc(saw) ─→ filter ─→ env(slow) ─→ out
                                                            context ─ctx→ voicer (re-spells degrees live)
                                            brightness ──map──m2s──→ filter.cutoff   (Good Button)
```

- The **`chord` operator** (new, ADR-0022) tracks the set of held root degrees. On a press of
  root degree `d` it emits the stacked scale-relative thirds `d, d+2, d+4` as `degree` Messages
  (and `d+6` when `size` = 4, a seventh); on release it emits the matching note-offs. It has **no
  context input** — it only does degree arithmetic. The downstream **Voicer** resolves each degree
  through the **`context`** node, so the same held degrees re-spell when the key changes. The third
  is scale-relative, so one operator spells major, minor, *and* diminished triads depending on
  where the root sits — the diatonic I–vii° set, always in key.
- The **chord buttons** are a new `control-surface` widget, `chord-button` (ADR-0022 tooling #7):
  a toggle whose payload is a custom `[degree, gate]` — a constant scale degree (the chord root)
  plus the button's gate, all seven hitting the one `/chord/set` address.

## Setup

1. **Run the instrument** (opens your default audio device, listens on UDP `0.0.0.0:9000`):

   ```sh
   cargo run -p reuben-native --bin reuben -- play instruments/chord-player.json
   ```

2. **Install Hexler TouchOSC** on a phone/tablet (or the desktop editor) — <https://hexler.net/touchosc>.

3. **Point TouchOSC at reuben.** Open `control-surfaces/chord-player.tosc`. The `.tosc` carries
   the layout + the OSC messages (addresses + ranges) but **not** a host: set the destination in
   TouchOSC's **connections** settings to the IP of the machine running reuben, port **9000**, send
   enabled. (The tablet and the reuben machine must share a network.)

> External OSC is block-quantized by design (it applies at the next block boundary) — honest, not
> a bug. Sample-accurate timing is the Clock's job, not a UDP datagram's.

Run everything below from the repo root.

## 1. Generate the surface

The instrument ships pre-annotated with `control` blocks, so the surface is a single `emit`.

**See what the surface will contain** (read-only):

```sh
python3 .claude/skills/control-surface/gen_surface.py infer instruments/chord-player.json
```

`infer` lists the **Brightness** Good Button (`/brightness`, `[0,1]`, rests at `0.5`) and every
node param with its resolved address — including `/context/root` (the key selector) and the
envelope's Attack/Release. (The seven chord buttons are *authored* `chord-button` controls, not
inferred candidates — they're already in the JSON.)

**Emit the surface:**

```sh
python3 .claude/skills/control-surface/gen_surface.py emit instruments/chord-player.json
# wrote control-surfaces/chord-player.tosc — 11 control(s), send to localhost:9000
```

Eleven controls: the **seven chord buttons** (I, ii, iii, IV, V, vi, vii — each a `chord-button`
firing `/chord/set [degree, gate]` with its fixed root degree 0..6), the **Key** fader
(`/context/root`, MIDI 48–72), the **Brightness** Good Button, and **Attack** + **Release** pad
faders.

## 2. Play it

**Tap chords.** With the instrument running and TouchOSC connected:

- Tap and **hold** the **I** button — a C-major triad (C E G) swells in on the slow-attack pad.
  Release it and the chord fades on the envelope's release.
- Walk a progression: hold **I**, then **V**, then **vi**, then **IV** — the classic pop loop, all
  diatonic and in key by construction. Each button is its own chord; you never play a wrong note.
- **Overlap** chords: hold **I** and **IV** at once (two fingers) — both sound; lift one finger and
  only that chord releases while the other keeps ringing. (The `chord` op tracks held roots
  independently; 12 voices cover four held triads.)
- **vii** is the diminished chord (vii°): the scale-relative third spells it automatically — proof
  the harmony lives in the context, not the button.

**Shape the pad.** Sweep **Brightness** `0 → 1` to open the filter from dull to bright; ride
**Attack**/**Release** for a faster or more washed pad.

**Change the key — the signature move.** Hold a chord (say **I**), then drag the **Key** fader up.
Watch (and hear) the *held* chord **re-spell live**: at root D it becomes a D triad, at root F an F
triad — the same I chord, transposed, with no re-press. Tap any chord button after moving the key
and it plays in the new key too. This live re-spell of *held* harmony is what the `context` bus
buys you, and the reason the Chord player is a Toy and not a static pad.

---

## What "passing" looks like

- Each chord button **sounds its triad** while held and **silences** on release; the pad attack
  and release feel like a pad.
- **Overlapping** chords sound together and **release independently** (lifting one finger doesn't
  cut the other chord).
- **vii** plays a diminished triad; **IV**/**V** major; **ii**/**iii**/**vi** minor — all without
  the player choosing intervals.
- Dragging **Key** while a chord is held **re-spells it live** to the new key (no re-press), and
  subsequent taps are in the new key.
- **Brightness** opens the filter; **Attack**/**Release** reshape the pad.

The automated proof behind this walkthrough lives in `cargo test -p reuben-core`: the `chord`
operator's oracle (`operators::chord`) and the end-to-end `chord_player` integration test (a
tapped triad sounds the three resolved chord tones, releases drop the right voices, two chords
release independently, a held chord re-spells on a key change, and the shipped instrument produces
audible output). The generator's `chord-button` widget is covered by
`python3 -m unittest` in `.claude/skills/control-surface/`.

The surface is disposable (ADR-0018): the instrument JSON is the source of truth; regenerate the
`.tosc` whenever the instrument changes.
